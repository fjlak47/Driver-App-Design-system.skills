# 设计规范摘要

本文件提炼自 `滴滴货运司机端设计规范.skills`，用于在实现页面和组件时快速查阅。

**重要说明**：本设计规范基于 `750px` 宽度画布制定，所有字号、间距、尺寸等参数均针对750px宽度画布优化，应用时应保持750px宽度画布（需自适应大小）。

**适配技巧**：
- 在body样式中设置`width: 100%; max-width: 750px; margin: 0 auto;`，确保页面在750px宽度下居中显示
- 移除所有子容器中的`max-width`和`margin: 0 auto;`属性，因为body已经控制了整体宽度
- 将媒体查询断点设置为`max-width: 750px`，确保在750px宽度时使用正常的样式，而不是小屏幕的样式

## 核心原则

- 清晰：信息层次分明，重点明确
- 高效：操作流程简洁，减少司机理解和操作成本
- 统一：颜色、字体、组件状态与结构保持一致

## 通用结构

### 页面与内容容器

- 页面容器使用移动端 `750px` 设计宽度语义
- 页面背景优先 `#F8F9FC`
- 内容区优先白底或浅灰分组，配合 `8px` 到 `32px` 圆角

### 卡片

- 基础卡片：白底、`8px` 到 `24px` 圆角、`24px` 内边距
- 带阴影卡片：白底 + 中等投影
- 无边框卡片：白底、无边框、`24px` 内边距、下间距 `16px`

### 弹窗

- 确认弹窗：宽 `600px`、白底、`16px` 圆角、`32px` 内边距、强投影
- 提示弹窗：宽 `400px`、白底、`16px` 圆角、`24px` 内边距、强投影
- 半弹窗：底部吸附、顶部圆角 `48px 48px 0 0`
